案例

我设计并交付的系统——在生产环境跑着。

以下系统由我作为某澳洲跨境电商集团 GM 兼联合创始人搭建。客户名与内部细节已匿名;架构、规模、做法是真的。

板块 01 · 自动化上架

6 万 SKU 货盘 → 定时上线的 listing

问题

一个多店运营,6 万+ SKU 的供应商货盘。选什么、写内容、定价全靠手工,慢,而且跨店根本同步不过来。

我搭了什么

一条 agent 管线:每周扫货盘、按适配度打分,再批量上架 Shopify,自动生成标题/描述/类目——每个都按成本底价定价,绝不亏本上架。

6 万+SKU 处理
每周自动选品扫描
一键批量上架带内容
板块 02 · SEO

22 个工具、按阶段编排的 SEO 工作台

问题

SEO 工作零散、不成体系:关键词和真实需求脱节,类目页没优化,没有可复用的月度节奏,AI 起草的内容还有上线不审的风险。

我搭了什么

一个工作台,22 个工具分 7 个阶段——关键词用真实搜索量验证、类目页优化、内容、外链、GBP、月报——发布前必须人工审批,不审不上线。

22工具
7工作阶段
审批制人工把关发布
板块 03 · 创意

产品图与广告素材按规格自动生成

问题

投放和上架老是等设计。每个广告位要不同比例,每个产品要白底、场景、细节图。设计成了瓶颈。

我搭了什么

一条自动出图管线:产品页成套图 + 广告素材按每个比例(1:1、1.91:1、9:16)生成,按模板统一品牌调性,标题/描述多版本随时可做 A/B 测试。

3+每图多个广告比例
统一模板化品牌调性
不等创意不再卡关键路径
板块 04 · 投放

9 个店铺的 Google Ads 实时监控

问题

一个经理账户下挂 9 个店,每天人工巡检才能抓住浪费;小问题(某条 campaign 崩了、零转化还在烧预算)藏到花了真金白银才发现。

我搭了什么

一块实时看板,直接从 Google Ads API 拉数据,4 桶异常检测——低 ROAS、低 CTR、零转化消耗、状态异常——自动标红。外加全套搭建:账户、GA4、GTM、GMC、Search 和 PMax,周度优化节奏。

9店铺一块看板
4桶异常自动标
实时数据直连 API
底层平台

100+ 工具,由 AI agent 编排

思路

四块都不是孤立的小脚本。它们跑在一个共享的内部平台上,100+ API 工具派发给 AI agent——所以上架、SEO、创意、投放用的是同一份数据、同一套护栏。
